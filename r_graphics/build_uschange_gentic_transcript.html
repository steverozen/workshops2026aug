## ggplot2 reproduction of a seaborn lineplot style, using uschange_mts.
##
## seaborn's lineplot aggregates replicates per x value into mean + 95% CI.
## uschange_mts has one observation per quarter, so we treat the 4 quarters
## within each year as replicates and aggregate to year-level mean + 95% CI.

library(DataSetsVerse)
library(timeSeriesDataSets)
library(tidyverse)

## Wide multivariate ts -> long data frame with year as the timepoint.
df <- as.data.frame(uschange_mts) |>
  mutate(
    date = as.Date(paste0(
      rep(1970:2016, each = 4)[seq_len(nrow(uschange_mts))], "-",
      c("01", "04", "07", "10")[((seq_len(nrow(uschange_mts)) - 1) %% 4) + 1],
      "-01"
    )),
    year = as.numeric(format(date, "%Y"))
  ) |>
  pivot_longer(
    cols = c(Consumption, Income, Production, Savings, Unemployment),
    names_to = "variable",
    values_to = "value"
  )

ggplot(df, aes(x = year, y = value,
               color = variable, linetype = variable, shape = variable)) +
  stat_summary(fun.data = mean_cl_normal, geom = "ribbon",
               aes(fill = variable), color = NA, alpha = 0.2) +
  stat_summary(fun = mean, geom = "line") +
  stat_summary(fun = mean, geom = "point", size = 2) +
  labs(x = "year", y = "value", color = "variable",
       linetype = "variable", shape = "variable", fill = "variable") +
  theme_gray() +
  theme(
    panel.background = element_rect(fill = "grey90"),
    panel.grid.major = element_line(color = "white"),
    panel.grid.minor = element_line(color = "white")
  )

what do you mean by: "The seaborn lineplot aggregates replicates per x value into mean + 95% CI ribbon."? how does that work?
assistant 12:40:20 Conversation size: 38,122 | [+] 6 input | ~322 output | 38,065 cache read | 51 cache write
When you call `sns.lineplot(x="timepoint", y="signal", data=fmri)`, seaborn looks at the data and notices that for each `timepoint` value there are **many rows** (multiple subjects × trials all measured at, say, timepoint=5). Rather than draw all those points, it:

1. **Groups** the rows by the x value (and by any `hue`/`style` grouping).
2. **Aggregates** within each group: computes the **mean** of `y` (that's the line/point).
3. **Estimates uncertainty** around that mean: by default a **95% confidence interval**, computed by bootstrapping the values within the group (resample with replacement 1000× by default, take the 2.5th/97.5th percentiles of the bootstrap means). That's the shaded ribbon.

In the fmri dataset, there are ~14 subjects, each measured at 19 timepoints in 2 regions × 2 events, so a single point like `(timepoint=5, region=parietal, event=stim)` has ~14 signal values. The dot is their mean, the ribbon is the bootstrap 95% CI of that mean.

**Why this matters for `uschange_mts`:** that dataset has exactly one observation per quarter, so per-x there are no replicates to aggregate, no mean to take, no CI to compute. That's why I grouped 4 quarters into a year, to manufacture 4 "replicates" per x value so the same mean+CI machinery has something to chew on. With `stat_summary(fun.data = mean_cl_normal, …)` in ggplot2 I used the normal-approximation CI rather than a bootstrap, but the idea is the same.

# View Current Session

Opens the current Claude Code session's JSONL transcript in [cclogviewer](https://github.com/Brads3290/cclogviewer), which renders it as an interactive HTML page and auto-opens it in the browser.

## How it works

1. **Resolve the project directory.** Claude Code stores session logs at
`~/.claude/projects/<project-dir>/<session-uuid>.jsonl`, where
`<project-dir>` is the current working directory with `/` replaced by `-`.
For example, `/home/steve/MEGA/ea` → `-home-steve-MEGA-ea`.

2. **Find the current session's .jsonl.** The current session is the most
recently modified `.jsonl` file in that project directory (Claude Code
appends to it constantly while the conversation is live).

3. **Run cclogviewer on it.** Without `-output`, cclogviewer writes to a temp
HTML file and auto-opens it in the default browser.

## Run the script

Run the helper script in the background (it opens a browser window and we
don't want to block on its output):

```bash
bash ~/.claude/skills/cclog-current-session/cclog-current-session.sh > /dev/null 2>&1 &
```
